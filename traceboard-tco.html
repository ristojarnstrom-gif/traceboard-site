03 — Where these numbers come from

An honest model, not a marketing number

This is an estimate, built from a documented licence price plus a transparent labour model — not a quote from either vendor. Adjust the assumptions to your own team's rates and the totals will move accordingly; the shape of the gap won't.

What's counted

  • TraceBoard Suite and ReqView figures use each vendor's own published, current pricing for a 15-user team.
  • ReqView's implementation estimate is priced at a €80/hr blended internal rate, for a light setup consistent with the "get started in minutes" onboarding its own users describe.
  • The "typical enterprise RM tool" column is not any single named product. It reflects a pattern visible across multiple public reviews of quote-only, enterprise-tier RM tools: licensing available only on request, and configuration described by their own users as time-consuming or requiring dedicated workshops.

What's not counted

  • Training time for end users beyond initial admin setup.
  • Any optional professional-services engagement a vendor might quote separately, which would generally increase — not decrease — the totals shown here.
  • Floating vs. named licensing, which for most tools in this category widens rather than narrows the gap.
  • Support and maintenance are already included in each vendor's published licence fee where one exists.
Pricing for enterprise-tier RM tools generally isn't published, so we haven't estimated a total for that column — only described the pattern their own reviewers consistently report. If you're evaluating one of these tools, ask the vendor directly for a written quote covering licence, implementation, and any ongoing configuration support, and use this table as a starting checklist for what to ask about.
04 — Fit assessment

When this gap actually matters

A lower total cost only matters if the cheaper tool also does the job. Here's where that's true — and where it isn't.

This TCO gap is the deciding factor if

  • Your team is 5–100 people and doesn't need a dedicated requirements-tooling administrator.
  • You want to be productive in hours or days, not after a multi-week configuration project.
  • Your data already lives in something simple — Excel, Word, or a lightweight Jira project — not a deeply customized enterprise workflow.
  • You'd rather your engineering budget go to engineering than to tool administration.

The more expensive tool may still be worth it if

  • You need Git/SVN-native requirements version control, or MBSE integration with Sparx EA or Capella (ReqView's strength).
  • You need a fully configurable rule engine and workflow automation that your own admins will tune and maintain for years.
  • You already have the in-house capacity to own a complex configuration long-term, and that flexibility is worth more to you than the setup cost.
  • Your process is genuinely non-standard enough that a pre-built workflow won't fit without deep customization.
